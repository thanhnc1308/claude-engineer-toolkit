# TypeScript Development Patterns

Comprehensive TypeScript knowledge base covering type safety, conventions, code review, refactoring, and error resolution.

## Reference Files

Detailed guidance is organized into reference files:

- **Type safety rules** — strict mode, type usage, type design, functions, error handling
  - **Reference:** `references/type-safety.md`
- **Conventions** — naming, file organization, imports, code style
  - **Reference:** `references/conventions.md`
- **Coding style** — immutability, error handling, input validation, console.log rules
  - **Reference:** `references/coding-style.md`
- **Hooks** — PostToolUse hooks (Prettier, tsc, console.log warning) and Stop hooks
  - **Reference:** `references/hooks.md`
- **API patterns** — API response format and common TypeScript/JavaScript patterns
  - **Reference:** `references/patterns.md`
- **Code review checklist** — type safety, type design, strict mode compliance, anti-patterns
  - **Reference:** `references/review-checklist.md`
- **Refactoring catalog** — 5 refactoring strategies (tighten types, extract types, improve generics, modernize patterns, improve DX)
  - **Reference:** `references/refactoring-catalog.md`
- **Error resolution** — 10 common TypeScript/build error patterns with fixes, minimal diff strategy
  - **Reference:** `references/error-resolution.md`
