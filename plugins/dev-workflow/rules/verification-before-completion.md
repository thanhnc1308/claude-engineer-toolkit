# Verification Before Completion

**NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE.**

Before claiming tests pass, build succeeds, work is complete, or expressing satisfaction:

1. **IDENTIFY** the verification command
2. **RUN** the full command (fresh, not cached)
3. **READ** complete output and check exit code
4. **VERIFY** output confirms the claim
5. **THEN** make the claim WITH evidence

Skip any step = unverified claim.

Red flags — STOP if you catch yourself:

- Using "should", "probably", "seems to"
- Expressing satisfaction before running verification
- About to commit/push/PR without verification
- Trusting subagent success reports without checking
- ANY wording implying success without having run verification
